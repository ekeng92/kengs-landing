# Keng's Landing Task Board Operating System

Purpose: make the board act like Eric's external memory, not another place he has to remember to check.

## Board lanes

- **Backlog** — useful someday, not committed.
- **To Do** — ready and worth doing soon.
- **In Progress** — actively being worked; keep this small.
- **Waiting / Blocked** — cannot move until someone/something responds.
- **Done** — completed and safe to forget.

## Task fields

- **Due / Follow-up** — when it should resurface, not always a hard deadline.
- **Energy** — `quick`, `deep`, `errand`, `call`.
- **Context** — `computer`, `phone`, `home`, `property`, `cpa`, `vendor`.
- **Blocked reason** — the thing that must happen before progress resumes.
- **Tags** — recommended: `tax`, `guest`, `ops`, `maintenance`, `listing`, `finance`, `product`, `receipt`, `launch`.

## AEON usage pattern

When Eric mentions a loose thread, convert it into a task with the best available fields. If unsure, capture it in Backlog rather than letting it disappear.

Weekly sweep:

- Move stale In Progress tasks back to To Do or Waiting.
- Surface Waiting tasks whose follow-up date is due.
- Keep no more than 3 active Keng's Landing tasks in In Progress.
- Prefer one digest over scattered nags.
